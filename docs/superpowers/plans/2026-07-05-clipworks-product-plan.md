## 1. 产品概述

### 1.1 一句话定位

**ClipWorks（映工厂）= 面向非技术用户的 AI 视频生成与剪辑工具。**

用户只需要输入一句话、上传一段素材，AI 自动完成脚本、分镜、剪辑、配音、出片。

### 1.2 核心用户

| 用户类型 | 痛点 | 使用场景 |
|---------|------|---------|
| 自媒体 / MCN | 做视频太耗时，不会 PR/AE | 口播精剪、短视频广告、产品种草 |
| 中小企业市场部 | 没有视频团队，外包贵 | 产品发布、功能更新、活动宣传 |
| 教育工作者 | 做课程视频门槛高 | 教程视频、课件动画 |
| 普通消费者 | 想给亲友做祝福/纪念视频 | 生日祝福、旅行回顾 |

### 1.3 核心价值主张

- **零门槛**：不需要懂剪辑、写代码
- **Agentic**：用自然语言反复修改，像和剪辑师对话
- **多引擎**：根据场景自动选择最合适渲染方案（模板 / 浏览器 / 浏览器自动化）
- **云+端**：云端重度渲染，浏览器端轻量编辑和预览

### 1.4 与竞品的差异化

| 产品 | 优势 | 劣势 | ClipWorks 差异 |
|-----|------|------|---------------|
| HeyGen | 数字人专业 | 贵、场景单一 | 不只做数字人，覆盖全视频工作流 |
| Runway | AI 效果强 | 学习曲线陡 | 面向小白，Agent 驱动 |
| Canva | 模板多 | 视频深度弱 | 更深的 AI 剪辑与自动分镜 |
| CapCut | 剪辑强 | 不够 AI | 从创意到成片全自动 |
| Remotion / HyperFrames / video-use | 能力强 | 只面向开发者 | 把开源能力打包成 GUI |

## 2. 商业模式

### 2.1 收入模式

**免费版（Freemium）**
- 每月 3 个视频
- 720p 输出
- 带 ClipWorks 水印
- 基础模板

**Pro 版：¥69/月 或 ¥599/年**
- 每月 50 个视频
- 1080p / 2K 输出
- 去水印
- 商用授权
- 优先渲染队列

**团队版：¥299/人/月**
- 无限视频（合理用量）
- 4K 输出
- 团队协作 / 品牌素材库
- API 接入
- 专属客服

**企业版：按用量议价**
- 私有部署 / 混合云
- 定制模型与模板
- SLA 保障

### 2.2 成本结构

| 成本项 | 说明 | 控制方式 |
|-------|------|---------|
| GPU/渲染 | 视频生成、AI 推理 | 队列 + 降级 + 用户分级 |
| 存储/CDN | 素材与成片存储分发 | 生命周期策略 + 对象存储 |
| LLM API | 脚本、分镜、修改建议 | 缓存 + 本地小模型 |
| 第三方授权 | 音乐、字体、素材 | 自建素材库 + 开源授权 |

### 2.3 关键假设

- 用户获取成本（CAC）< 12 个月用户价值（LTV）
- 免费用户转化率 > 3%
- Pro 用户月均渲染 15 条以上

## 3. 产品功能路线图

### 3.1 当前原型状态

已具备：
- ✅ 首页 Agent Launchpad
- ✅ 项目库 / 工作台 / 素材库 / 设置等页面
- ✅ 后端 API（项目、渲染 Job、素材）
- ✅ 多引擎渲染架构（HyperFrames / Remotion / video-use / Mock）
- ✅ Docker Compose 部署
- ⚠️ 视频输出为假样本（MockProvider）
- ⚠️ 素材系统为骨架
- ⚠️ 剪辑能力仅展示
- ⚠️ Agent 只支持单次生成，不能多轮修改

### 3.2 P0：真实出片（2-4 周）

**目标**：输入一句话，真实输出 MP4。

**关键功能**：
1. 修复并稳定至少一条真实渲染链路（建议先 Remotion 或 HyperFrames）
2. 渲染任务异步队列（Celery / RQ / 自建 worker）
3. 渲染进度实时推送（WebSocket 或 SSE）
4. 输出视频存储到本地 / 对象存储
5. 前端预览真实视频（而非 HTML 预览）
6. 失败重试与降级策略

**验收标准**：
- 创建项目 → 输入 prompt → 5 分钟内拿到真实 MP4
- 渲染失败自动重试一次，并返回友好错误

### 3.3 P1：真实素材系统（2-3 周）

**目标**：用户可以上传、管理、预览素材。

**关键功能**：
1. 分块上传 / 断点续传
2. 视频转码（多分辨率：360p/720p/1080p）
3. 自动生成缩略图和封面
4. 素材元数据提取（时长、分辨率、fps、codec）
5. 素材标签 / 搜索
6. 删除与生命周期管理

**验收标准**：
- 上传 500MB 视频稳定完成
- 素材库页面可预览所有素材

### 3.4 P2：可交互剪辑（3-6 周）

**目标**：时间轴可拖拽编辑，预览实时同步。

**关键功能**：
1. 轨道拖拽排序、裁剪、分割、删除
2. Clip 时长/位置/层级调整
3. 实时预览（当前 HTML 预览升级）
4. 字幕编辑与样式调整
5. 音轨对齐与音量调节
6. 撤销 / 重做

**验收标准**：
- 用户能在 2 分钟内完成一个简单剪辑调整
- 修改后重新渲染，输出符合预期

### 3.5 P3：Agent 迭代（2-4 周）

**目标**：用户可以用自然语言反复修改视频。

**关键功能**：
1. 多轮对话上下文保持
2. 局部修改（只改某个 scene / clip）
3. 修改建议（“这里加点音乐”“换个风格”）
4. 版本对比与回退
5. 用户反馈强化学习（可选）

**验收标准**：
- 用户说“把第三句字幕改成红色”，30 秒内生效

### 3.6 P4：商业化（4-8 周）

**目标**：能收费、能协作、能稳定运行。

**关键功能**：
1. 真实登录（OAuth / 邮箱 / 手机）
2. 支付订阅（Stripe / 支付宝 / 微信）
3. 用量配额与限流
4. 团队协作（成员、权限、共享素材库）
5. 水印与商用授权
6. 监控、日志、告警
7. CI/CD、灰度发布

## 4. 技术架构

### 4.1 当前技术栈

| 层级 | 技术 |
|-----|------|
| 前端 | Next.js 14 + React 18 + Tailwind CSS + Zustand |
| 后端 | FastAPI + SQLAlchemy + PostgreSQL + Redis |
| 渲染 | HyperFrames + Remotion + video-use (browser-use) |
| 部署 | Docker Compose |
| AI | Kimi / OpenAI API |

### 4.3 关键模块设计

#### 4.3.1 渲染引擎选择器

根据输入自动选择引擎：

| 场景 | 推荐引擎 | 原因 |
|-----|---------|------|
| 文本/图片动画、模板视频 | Remotion | 代码驱动，可控性强 |
| 网页动画、HTML 转视频 | HyperFrames | 直接渲染 DOM |
| 已有素材剪辑、口播 | video-use | 浏览器自动化编辑 |
| 快速预览 / 失败兜底 | Mock / HTML 预览 | 低成本 |

#### 4.3.2 渲染队列

- 使用 Celery + Redis 或 RQ
- 任务状态：`queued` → `running` → `completed` / `failed`
- 长任务超时：10 分钟
- 失败重试：最多 2 次
- 进度推送：WebSocket 或 Server-Sent Events

#### 4.3.3 存储策略

| 数据类型 | 存储 | 生命周期 |
|---------|------|---------|
| 用户上传素材 | S3 / MinIO / OSS | 项目删除后 30 天清理 |
| 成片 | S3 + CDN | 免费 7 天，付费长期 |
| 预览 HTML | 本地 / S3 | 与项目同步 |
| 缩略图 | S3 + CDN | 与素材同步 |
| 数据库 | PostgreSQL | 持久化 |

#### 4.3.4 AI Agent 架构

```
User Prompt
    │
    ▼
┌───────────────┐
│ Intent Parser │  解析用户意图：创建 / 修改 / 导出
└───────┬───────┘
        │
        ▼
┌───────────────┐
│ Planner       │  生成 / 修改 composition
└───────┬───────┘
        │
        ▼
┌───────────────┐
│ Executor      │  调用素材、渲染、返回结果
└───────────────┘
```

## 6. 风险与应对

| 风险 | 影响 | 应对 |
|-----|------|------|
| 渲染成本过高 | 盈利困难 | 用户分级、队列限速、云端 GPU 按需、本地渲染选项 |
| 渲染质量不稳定 | 用户流失 | 多引擎 fallback、人工审核模板、A/B 测试 |
| LLM 输出不可控 | 剪辑结果差 | prompt engineering、结构化输出、用户可手动修正 |
| 竞品快速迭代 | 差异化被稀释 | 聚焦“小白用户 + Agent 对话”细分场景 |
| 版权问题 | 法律风险 | 自建素材库、用户上传合规声明、商用授权分层 |
| 技术债务 | 迭代变慢 | 每阶段结束重构 + 自动化测试覆盖 > 70% |

## 8. 下一步建议

如果你决定推进，建议**立刻启动 P0：真实出片**。这是最核心的里程碑，也是所有后续功能的前提。

### P0 第一周的 3 件事

1. **确定主渲染引擎**：建议在 Remotion 和 HyperFrames 之间选一个先跑通真实 MP4
2. **搭建渲染队列**：用 Redis + Celery 把同步渲染改成异步任务
3. **前端接真实视频**：预览区从 HTML 切换到 `<video>`，支持下载

完成这 3 件事后，ClipWorks 就从“原型”变成“能用的产品 Demo”。
