# ClipWorks Agentic UI 设计文档

> 版本：2026-07-05  
> 状态：待用户确认后进入实现  
> 关联设计稿：`2026-07-05-clipworks-agentic-ui-concepts.html`

---

## 1. 背景与目标

当前 ClipWorks 的前端界面更像「传统编辑器 + 侧边栏聊天助手」，Agent 的存在感不强。用户需要看到一个真正以 AI Agent 为核心的视频创作界面：

- 小白用户能用一句话开始创作；
- 生成后能用自然语言修改具体场景；
- 整个创作过程能感知 AI 在做什么；
- 同时保留对画面的精准控制。

---

## 2. 用户确认的设计方向

| 维度 | 选择 |
|------|------|
| 交互模型 | **Hybrid 混合式** — 对话为主，节点精准控制 |
| 修改锚点 | **场景卡片** — 点画面 → 说修改 |
| 首页入口 | **Agent Launchpad** — 一句话开始创作 |
| 工作区布局 | **三栏影院式** — 左 Agent/场景，中预览，右属性 |
| 生成反馈 | **可视化流水线** — 像工厂流水线展示每个环节 |

---

## 3. 设计原则

1. **Agent 优先**：界面围绕「与 AI 协作」设计，而非把 AI 塞到传统编辑器里。
2. **画面即控制**：用户看到场景画面就能直接改，不需要懂时间线。
3. **过程可见**：生成过程像工厂流水线，每个环节状态清晰。
4. **渐进披露**：默认简单，高级功能（时间线、精细属性）可展开或切换到专业模式。
5. **品牌一致**：延续 ClipWorks 暗色电影感设计系统，强化「映工厂」心智。

## 4. 页面与组件设计

### 4.1 首页：Agent Launchpad

**目标**：让用户 0 思考开始创作。

**布局**：
- 顶部极简导航：Logo + 创作/项目库/素材库/设置 + 用户头像
- 中央大标题：「一句话，一段素材，一条成片」
- 核心输入框：大号圆角输入框 +「开始创作」按钮
- 热门模板 chips：小红书口播、SaaS 产品发布、教程视频、短视频广告
- 底部最近项目条：横向滚动卡片，展示 3-5 个最近项目

**交互**：
- 输入框支持回车直接提交；
- 点击热门 chip 自动填充输入框；
- 提交后创建项目并跳转到工作台；
- 最近项目条点击直接进入对应项目。

### 4.2 工作台：三栏影院式布局

**整体结构**：

```
┌─────────────────────────────────────────────────────────────┐
│  TopBar: 返回 | 项目标题 [状态]        导出 HTML | 导出 MP4  │
├───────────────┬──────────────────────────────┬──────────────┤
│               │                              │              │
│  左栏 320px   │      中间 预览区 (flex-1)     │  右栏 260px  │
│  - Agent 对话  │      - 视频画面              │  - 属性面板   │
│  - 场景卡片   │      - 播放控制条            │  - 素材列表   │
│               │                              │              │
└───────────────┴──────────────────────────────┴──────────────┘
```

#### 4.2.1 左栏

**上半部分：Agent 对话**
- 头像 +「AI 导演」标题
- 消息气泡：Agent 在左，用户在右
- 快捷提示 chips：更活泼一点、缩短到 15 秒、换背景音乐
- 输入框 + 发送按钮

**下半部分：场景卡片**
- 标题「场景卡片」+ 提示「点卡片修改」
- 每个场景卡片包含：
  - 左侧缩略图（渐变色占位或真实帧）
  - 场景名称 + 时间区间
  - 右侧编辑按钮
- 当前选中的场景卡片高亮（brand 边框）
- 点击卡片 = 选中该场景，可在对话中说「改这个…」
- 点击编辑按钮 = 弹出/展开该场景的属性编辑

#### 4.2.2 中间：预览区

- 深色背景，视频居中
- 默认显示 16:9 画幅，支持切换 9:16 / 1:1
- 底部播放控制条：播放/暂停、进度条、当前时间/总时长
- 生成中/修改中时，预览区上方可叠加流水线 mini 版

#### 4.2.3 右栏：属性与素材

**属性面板**：
- 标题输入
- 主色调选择
- 时长滑块
- 画幅切换（16:9 / 9:16 / 1:1）

**素材面板**：
- 当前项目用到的图片、音乐、视频素材列表
- 每个素材显示缩略图 + 文件名 + 类型

### 4.3 场景卡片系统

场景卡片是「Hybrid」交互的核心锚点。

**数据结构**（每个场景）：
```json
{
  "id": "scene-1",
  "index": 0,
  "name": "产品痛点引入",
  "description": "用一个问题引出用户痛点",
  "start_time": 0,
  "duration": 8,
  "thumbnail": "...",
  "text_clips": [...],
  "visual_clips": [...]
}
```

**交互状态**：
- 默认：正常边框
- hover：边框变亮、轻微上浮
- 选中：brand 色边框 + 微光
- 生成中：骨架屏或脉冲动画

**AI 编辑入口**：
- 场景卡片右上角编辑按钮
- 点击后在 Agent 对话中自动带上上下文：「正在编辑「产品痛点引入」（0:00-0:08），你想怎么改？」

### 4.4 可视化流水线

**目标**：让用户感知 AI 在做什么，同时强化「映工厂」品牌。

**节点设计**（水平排列）：
1. 理解需求
2. 分析素材
3. 编写脚本
4. 生成场景
5. 渲染成片
6. 输出成片

**状态样式**：
- 已完成：绿色边框 + 对勾
- 进行中：brand 色边框 + 脉冲动画
- 未开始：灰色边框
- 节点之间用连线连接，已完成/进行中段高亮

**位置**：
- 生成时：工作台中间预览区下方出现一条流水线状态条
- 也可在生成完成后的详情页展示完整流水线回放

**文案**：
- 每个节点下方显示一句话当前动作，例如：
  - 生成场景中：「正在生成场景 3/4：用户证言」

## 5. 核心交互流程

### 5.1 从零创建视频

1. 用户在首页 Launchpad 输入：「帮我做一个 30 秒的产品介绍视频」
2. 系统创建项目，跳转到工作台
3. 工作台自动进入生成状态，展示流水线
4. Agent 完成：理解需求 → 分析素材 → 写脚本 → 生成场景 → 渲染
5. 成片在预览区播放，场景卡片列表出现
6. 用户可以在对话中继续修改，或点击场景卡片精确编辑

### 5.2 修改某个场景

1. 用户点击左侧「场景 2：解决方案展示」卡片
2. 卡片高亮，Agent 对话自动追加：「你选中了「解决方案展示」（0:08-0:18），想怎么改？」
3. 用户输入：「把背景换成深蓝色的科技感」
4. Agent 仅重新生成该场景，流水线显示局部更新
5. 预览区自动跳转到该场景，展示修改结果

### 5.3 全局修改

1. 用户在对话中输入：「整体缩短到 15 秒」
2. Agent 重新规划所有场景时长
3. 流水线重新跑「编写脚本 → 生成场景 → 渲染成片」
4. 场景卡片时间区间更新

### 6.2 数据结构扩展

后端 `Composition` 已经包含 `tracks` 和 `clips`。场景卡片可以通过对 clips 按时间分组生成：

```typescript
interface Scene {
  id: string;
  index: number;
  name: string;
  description?: string;
  start_time: number;
  duration: number;
  thumbnail?: string;
  text_content?: string;
  visual_content?: string;
}
```

场景可以在后端生成（LLM 规划时输出场景列表）或前端根据 clips 的时间区间自动分组。

## 8. 视觉规范

延续 `docs/design/clipworks-design.md` 的设计系统：

- 深色模式优先：`--bg-base: #0a0a0f`
- 品牌色：`--brand-500: #0ea5e9`
- 成功/进行中：`--success: #22c55e`，`--warning: #f59e0b`
- 圆角：`--radius-md: 10px`，`--radius-lg: 16px`
- 字体：`Inter` + `Noto Sans SC`
- 卡片 hover：`translateY(-2px)` + 边框高亮

新增元素：
- 流水线节点：圆形，直径 64px，边框 2px
- 场景卡片：横向卡片，高度 64px，缩略图 80×48px
- Launchpad 输入框：圆角 16px，brand glow on focus
